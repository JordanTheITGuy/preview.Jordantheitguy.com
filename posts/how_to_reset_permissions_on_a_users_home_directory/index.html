How To Reset Permissions on a Users Home Directory

Jordan Benzing
 2015-06-24  382 words   2 minutes 
Contents

Below is a script, provided as is, that is a non-fancy way to reset permissions with some sense of logic on a users home directory. I’ve got a slightly better version of it, but I want to tweak a few things before posting it online.

 0
 1$Folder = Read-Host "Enter the name of the user to reset permissions for"
 2#Enter the name of the user directory here
 3$Existance = Get-ADUser -LDAPFilter "(SAMAccountName=$Folder)"
 4$Path = '\\YOURNETWORKSHAREHERE'
 5$Fullpath = $Path + $Folder
 6#Combine the network path with the variable of the users home folder
 7$CheckExistance = Test-Path $fullpath
 8#Checks to make sure the path exists
 9If ($Existance -eq $Null) {
10    "User Does not exist in AD"
11    #If you get this, the ldap filter didnt find the user in active directory.
12}
13ElseIf ($CheckExistance -Eq 'True' ){
14    $Confirmation = Read-Host "Are you sure you want to edit permissions for " $Folder " enter Y to continue anything else to exit"
15    If ($Confirmation -Eq 'Y') {
16        $User = 'YOURDOMAINHERE\' + $Folder
17        $Admins = 'YOURDOMAINHERE\Domain Admins'
18        $HomeShareManagers = 'YOURDOMAINHERE\WHATEVEROTHERGROUPYOUWANT'
19        $Path = $Path + $Folder
20        icacls "$Path" /setowner ("$User") /Q /T /C
21        #Sets the owner of the root user folder and all sub-folders to the current "for" user.
22        iCacls "$Path" /Q /C /T /reset
23        #Resets the all folders and files starting at the root to ONLY have the permissions granted by inheritance
24        icacls "$Path" /Grant :R ("$User" + ':(OI)(CI)M') /Q
25        #Grants the user in question modify rights to all objects within the root user folder.
26        icacls "$Path" /Grant :R ("$Admins" + ':(OI)(CI)F') /Q
27        #Grants the Domain Admins group Full control rights to all objects within the root user folder.
28        icacls "$Path" /Grant :R ("$HomeShareManagers" + ':(OI)(CI)F') /Q
29        #Grants the site specific Home Share Managers group Full Control within the root user folder.
30        icacls "$path" /Remove Builtin /Q /T
31        #Removes the builtin user ACL. This is auto generated and applied when a /reset is performed
32        iCacls "$Path" /inheritance:R
33        #Turns off inheritance at the root of the user folder.
34    }
35}
36Else {
37    'Something Went Wrong'
38    #Chances are in this case the folder is not named the same as the user's Login ID
39}
